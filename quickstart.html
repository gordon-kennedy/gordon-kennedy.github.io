    QuickStart for MAHLE Internal Wikiüîó

    A set of sequential self contained minimal steps which walk the user quickly through understanding and contributing to a wiki from cold. For speed and minimality these steps use an existing wiki (the one containing this page), and don't use VSCode. They also assume python and git have already been installed. See subsequent pages for fuller information on creating new wikis and/or using VSCode for additional convenience.

    1. clone MAHLE repo using gitüîó

    destination = "C:\Local Copy\DocDocumentationMake\Controlled"

    2. create virtual environment with pythonüîó

    Create a new python virtual environment with a name of your choice (here ".venv") in a folder of your choice, then activate the new virtual environment and list installed packages to see an empty list.
    > python -m venv .venv
    > "C:\Users\kennedgo\Documents\mypython8\.venv\Scripts\activate.bat"
    > pip list

    3. install mkdocs and mkdocs-materialüîó

    Install the python packages "mkdocs" and "mkdocs-material" and then list installed packages again to see all the new items installed.
    > python -m pip install mkdocs
    > python -m pip install mkdocs-material
    > pip list

    B. session steps for using the platformüîó

    1. open command terminal sessionüîó

    Windows Key + R "cmd" and click OK

    2. activate python virtual environmentüîó

    > "C:\Users\kennedgo\Documents\mypython7\.venv\Scripts\activate.bat"
    Observe that the command prompt is now prefixed by the name of the virtual environment.

    3. start serving website locallyüîó

    Change to a locally cloned mkdocs folder and launch the local mkdocs web server
    (.venv)> cd "C:\Local Copy\DocDocumentationMake\Controlled"
    (.venv)> mkdocs serve
    Click on hyperlink to see this page in a web browser at "http://127.0.0.1:8000/issues/quickstart.html"

    4. edit markdownüîó

    Use a text editor of your choice (eg. Notepad++ or VSCode) to edit one of the .md files.
    Observe that the page in the web browser updates every time you save any changes.

    5. stop serving website locallyüîó

    Ctrl Key + C

    6. build website locallyüîó

    (.venv)> mkdocs build
    Observe that a local folder called "published_website" appears (or is overwritten).

    7. deactivate python venvüîó

    (.venv)> "C:\Users\kennedgo\Documents\mypython7\.venv\Scripts\deactivate.bat"
    Observe that the command prompt prefix disappears.

    8. use local build folder to overwrite shared websiteüîó

    > copy_published_site_to_server.bat

    9. close command terminal sessionüîó

    > exit
